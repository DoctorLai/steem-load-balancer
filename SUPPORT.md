# Support

## Getting Help

Before opening a support request:

1. Check the [README](README.md), especially Configuration and Troubleshooting.
2. Search [existing issues](https://github.com/DoctorLai/steem-load-balancer/issues).
3. Confirm the problem still occurs on a supported Node.js version with the
   latest `main` branch or Docker image.

Use [GitHub Discussions](https://github.com/DoctorLai/steem-load-balancer/discussions)
for setup questions and operational advice. Use a
[GitHub issue](https://github.com/DoctorLai/steem-load-balancer/issues/new/choose)
for reproducible defects or feature requests.

Include the application version, Node.js or Docker version, relevant sanitized
configuration, logs, expected behavior, and reproduction steps. Never include
API keys, private headers, certificates, or other credentials.

Security vulnerabilities must be reported privately as described in
[SECURITY.md](SECURITY.md).

## Support Scope

Community support is best effort. Maintainers cannot provide guarantees for
third-party RPC availability, custom infrastructure, or modified deployments.
